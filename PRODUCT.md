# Product

## Register

brand

## Users

Gleam and BEAM developers evaluating Fluid protocol infrastructure. They are likely comparing whether spillway can give their collaborative systems a smaller, typed core for sequencing, sessions, signals, summaries, nacks, and JWT validation without pulling in a full server stack.

## Product Purpose

Spillway is a type-safe Gleam library for Fluid Framework protocol logic. The website should make the library feel trustworthy, focused, and easy to place in the wider Fluid stack: a precise component for controlling collaborative document flow rather than a generic app platform.

## Brand Personality

Hydraulic, precise, calm. The voice should feel like an engineering diagram for water control: controlled pressure, reliable gates, clean channels, and measured flow.

## Anti-references

Avoid generic SaaS landing pages with repeated card grids, fake metrics, gradient-text headlines, and stock "developer productivity" tropes. Avoid terminal cosplay, Microsoft/Fluid brand cloning, and playful water cartoons.

## Design Principles

- Make flow legible: explain protocol movement as controlled stages, not abstract feature claims.
- Show typed confidence: use real API names and small code excerpts instead of vague marketing copy.
- Stay calm under pressure: communicate reliability through restraint, spacing, and clear hierarchy.
- Be stack-aware: position spillway as one layer in a larger water-themed system without requiring that context.
- Prefer signal over spectacle: every visual device should clarify sequencing, sessions, or boundaries.

## Accessibility & Inclusion

Target WCAG AA or better, with readable contrast for body and muted text, keyboard-friendly links, and reduced-motion alternatives for every animation.
